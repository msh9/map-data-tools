# shared-configuration

Static configuration files consumed by multiple tilemaker utilities. These files are committed to the repository and updated infrequently.

## Contents

### `faa-chart-flyer-region-mapping.json`

Maps geographic region names to FAA chart directory names, structured as `{chart_type} -> {region} -> [chart_list]`. Consumed by `faa-raster-chart-preprocess` when building regional VRT mosaics.

### `sectional-chart-masks/`

One GeoJSON file per FAA sectional chart describing its coverage polygon. Used by `faa-raster-chart-preprocess` to clip sectional COG outputs. Files were generated by `faa-static-data/convert_faa_sectional_coverage.py` from `faa-sectional-coverage.txt` and then manually QA'd and corrected in QGIS for several charts (Las Vegas, Denver, Los Angeles, Hawaiian Islands, and others).

### `terminal-area-fly-chart-masks/`

One GeoJSON file per FAA terminal area chart location (naming format: `{Location}_TAC.geojson`). TAC and paired FLY charts share the same mask. These masks are hand-drawn in GIS — the FAA does not publish formal coverage specifications for TAC charts. Consumed by `faa-raster-chart-preprocess` for TAC and FLY clipped COG outputs.

### `mountain-regions-masks/`

GeoJSON polygon definitions for mountainous regions of interest (currently CONUS). Consumed by `dem-2-slope`, and `sentinel-mosaic` to clip processed outputs to areas of interest.

### `usgs-national-map-file-lists/`

Text files with one USGS 3DEP 1-arc-second DEM URL per line, generated by the [USGS National Map Downloader](https://apps.nationalmap.gov/downloader/). Includes per-region lists and a combined list (`usgs-1-arc-second-dem-generated-mountainous-regions.txt`). Consumed by `dem-2-slope`.

### `faa-sectional-coverage.txt`

Source data transcribed from the FAA VFR charts specification manual coverage table. Input to `faa-static-data/convert_faa_sectional_coverage.py`, which produces the GeoJSON masks in `sectional-chart-masks/`.
